# Design Pillars

## 1. Learn Through Helpful Play

English practice should come from helping characters solve grounded travel problems, not from disconnected quizzes.

## 2. Vietnam as a Living Classroom

Each region should teach language through place: landscapes, food, customs, transportation, weather, and local routines.

## 3. Short, Repeatable Dialogue

Lessons should be small enough to replay. Each dialogue should have a clear learning goal and a practical phrase set.

## 4. Data-Driven Expansion

Core systems should read structured content from `data/` so designers can add locations, NPCs, quests, and lessons without touching gameplay code.

## 5. Confidence Over Punishment

The tone should be encouraging. Mistakes should lead to hints, retries, and passport progress rather than failure screens.

## 6. No Advanced AI in MVP

The MVP should use authored content only. AI-assisted dialogue, adaptive tutoring, or procedural conversations can be explored after the core loop is proven.
