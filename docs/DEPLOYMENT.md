# Deployment Notes

These notes prepare the Phaser web MVP for external playtesting as a static Vite build. This project does **not** deploy automatically from this PR.

## Local Production Preview

Build the production bundle:

```sh
npm run build
```

Preview the built site locally:

```sh
npm run preview
```

Vite serves the production build from `dist/` during preview. Use this flow before sending a test link so the build output is checked separately from the development server.

## Build Output

- `dist/` is the generated static site output from `npm run build`.
- `dist/` can be uploaded to static hosts or generated by the host during deployment.
- `dist/` should not be committed to git.

## Do Not Commit

Do not commit generated or dependency folders:

- `dist/`
- `node_modules/`
- `build/`

These are already ignored by `.gitignore`.

## Vercel Notes

Vercel can deploy the Vite app with the default static build settings:

- Framework preset: `Vite` if detected.
- Install command: `npm install` or the Vercel default.
- Build command: `npm run build`.
- Output directory: `dist`.

No hardcoded Vite `base` path is needed for a normal Vercel project domain or custom domain.

## Netlify Notes

Netlify can deploy the Vite app as a static site:

- Build command: `npm run build`.
- Publish directory: `dist`.
- Local preview before pushing: `npm run build` then `npm run preview`.

No hardcoded Vite `base` path is needed for the default Netlify site URL or a custom domain.

## GitHub Pages Notes

GitHub Pages can host the static `dist/` output, but repository pages usually need a configured base path when the site is served from a subpath such as:

```txt
https://<user>.github.io/<repo>/
```

For that case, configure Vite with a matching base path, for example:

```ts
export default defineConfig({
  base: '/english-vietnam-game-rpg/',
});
```

Do not set this base path for Vercel, Netlify, custom domains, or local-only preview unless the deployment target actually serves the site from that subpath.

## Manual Deployment Checklist

1. Run `npm run typecheck`.
2. Run `python tools/validate_content.py`.
3. Run `npm run build`.
4. Run `npm run preview` and open the preview URL.
5. Confirm the Ha Giang scene loads.
6. Confirm player movement works.
7. Confirm NPC interaction works.
8. Confirm DialogueBox, quest HUD, and reward toast feedback still work.
9. Deploy using the selected static host only after the local production preview is acceptable.
